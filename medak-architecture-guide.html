The Two-Agent Architecture

The core innovation: two Gemini 2.0 Flash Live agents share a Redis-backed EmergencySnapshot and coordinate through it — not by talking to each other directly.

How the agents coordinate (without talking to each other)

The User Agent writes fields to the EmergencySnapshot (emergency_type, consciousness, breathing, etc.). The Dispatch Agent reads from it. When the 112 operator asks a question the Dispatch Agent can't answer, it writes the question to dispatch_questions[] in the snapshot. The User Agent polls for pending questions, asks the user (or infers from environment), and writes the answer to ua_answers[]. The Dispatch Agent picks it up and relays to the operator. Target round-trip: < 5 seconds.

Frontend Services

The lib/ folder contains the real engine — WebSocket management, audio/video capture, GPS, and sensor monitoring.

WebSocket lib/websocket.ts

Client → Server: audio chunks (base64 PCM), video frames (base64 JPEG), user tap/text responses, ping keepalives (15s).

Server → Client: transcripts, phase/confidence updates, user questions, RESOLVED (with ETA), FAILED, pong.

Auto-reconnect with exponential backoff (2s, 4s, 8s), max 3 attempts.

Audio Capture lib/audio.ts

16kHz mono PCM via expo-av. Simulates streaming with sequential 250ms recordings: prepare → start → wait → stop → base64 encode → send via WS → delete file. Repeat.

Camera Capture lib/camera.ts

Hidden 1×1px CameraView capturing frames every 750ms (~1.3fps). JPEG quality 0.3 for bandwidth. Sent as base64 via WebSocket. Used for scene analysis (fire, accident, etc.).

Danger Detection lib/useDangerDetection.ts

Monitors accelerometer via expo-sensors. Detects rapid Z-axis drop (fall) and high-frequency vibration (shake SOS). Three sensitivity levels. Triggers alarm screen with countdown.

SOS Flow lib/sosFlow.ts

Orchestrates the trigger: requests GPS + user_id + device_id in parallel → fires POST /api/sos + reverse geocode simultaneously (address never blocks SOS) → returns session_id.

Location lib/location.ts

GPS with 10s timeout (balanced accuracy). Reverse geocoding with 2s timeout — extracts street + number + city. Always non-blocking: SOS fires even without address.

The Two Agents

User Agent user_agent.py

Gemini 2.0 Flash Live session. Passive triage — observes mic + camera, never requires user input. Speaks Serbian.

8 Tool Calls

  • confirm_location(address) — marks location confirmed (+0.35)
  • set_emergency_type(type) — biggest confidence jump (+0.25)
  • set_clinical_fields(conscious?, breathing?, count?)
  • append_free_text(utterance) — raw user speech for context
  • get_pending_dispatch_question() — polls for operator Qs
  • answer_dispatch_question(q, a) — replies to operator
  • surface_user_question(q) — shows YES/NO on UI (max 1, auto-dismiss 5s)
  • record_user_input(type, value) — stores tap/text from UI
Gemini Live · TEXT mode

Dispatch Agent dispatch_agent.py

Gemini 2.0 Flash Live session. Speaks to 112 operator via Twilio VoIP. First sentence always identifies as automated relay.

5 Tool Calls

  • get_emergency_brief() — reads snapshot, returns formatted brief
  • queue_question_for_user(q) — asks user via User Agent
  • get_user_answer(q) — polls for answer (returns PENDING if waiting)
  • update_call_status(status) — DIALING→CONNECTED→DROPPED
  • confirm_dispatch(eta) — CONFIRMED + ETA → triggers RESOLVED
Gemini Live · AUDIO+TEXT mode

Failure & Retry Logic

Call Failures

If call_status becomes DROPPED, orchestrator retries the Dispatch Agent up to 3 times with exponential backoff: 2s → 4s → 8s. Each retry creates a fresh Gemini Live session and re-reads the snapshot.

Final Fallback

After 3 failed attempts: phase → FAILED. System sends SMS with GPS coordinates + emergency type to 112. Frontend shows: "Automatski poziv nije uspeo. Zamolite nekoga da pozove 112."
